## Why testing matters

Slopeside is designed to improve Meta Ads performance by strengthening audience signals.

The only way to fairly evaluate that impact is through a controlled test.

Testing removes guesswork and ensures you’re measuring the audience itself, not changes in creative, budget, or structure.

---

## Why A/B testing is recommended

Without a clean test, it’s impossible to know what caused a performance change.

A proper A/B test allows you to:
- Compare Slopeside audiences against your existing audiences
- Isolate the impact of audience quality
- Make confident decisions based on real data

This approach protects you from false positives and false negatives.

---

## What testing is not

Testing is not:
- Launching a new campaign with multiple changes
- Comparing results across different creatives
- Switching audiences mid-flight
- Expecting instant results without a learning phase

These approaches make results unreliable.

---

## When you should test

You should test Slopeside when:
- You already have a campaign that performs reasonably well
- You want to evaluate a new audience cleanly
- You’re planning to scale and want more predictable inputs

If you don’t have a baseline campaign, consider launching one before testing.

---

## Common questions

**Do I need to pause my existing campaigns?**  
No. Testing can run in parallel without disrupting current performance.

**How long should a test run?**  
Tests should run long enough for Meta to learn and stabilize. Avoid drawing conclusions too early.
